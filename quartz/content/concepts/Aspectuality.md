# Aspectuality

> Every representation selects a relevant subset of properties from its target — a particular aspect — and by doing so already presupposes relevance realization; this means representations cannot be the source of RR, only its output.

## Vervaeke's framing
In Ep 29, Vervaeke introduces aspectuality to test whether representations can generate relevance realization. Any representation must represent its target *under some aspect* — some selected subset of properties. The representation of a cup as "graspable" selects some properties (shape, size, handle) and ignores others (color, history, material composition). This selection is already a relevance judgment — it presupposes that the selected properties are the relevant ones for some purpose. Therefore, representations cannot generate relevance realization; they already depend on it. The same argument applies to any theory that takes representation as primitive (Ep 29).

**The argument structure**: The argument is a circularity demonstration: if you try to explain RR in terms of representations, you discover that every representation is already aspectual — it already embeds a relevance judgment. So representations presuppose RR rather than generating it. This is one of three tests (along with computation-via-inference and modularity) that fail, pointing Vervaeke toward the conclusion that RR must be grounded in participatory knowing, not in any representational structure (Ep 29).

**Aspectuality and FINSTING**: The positive account that Vervaeke develops in response: the mind's capacity to track "this here now" (FINSTING) is pre-aspectual and pre-representational — it is enacted, participatory, and prior to the selection of any particular aspect. FINSTING is what generates the initial salience tag from which aspectual representations can then be built. Aspectuality is thus a higher-level property that depends on the more fundamental enactive demonstrative reference (Ep 29).
